# LesionIQ Research — Semester Project

This folder is the **research project**, completely separate from the
hackathon code in `backend/` and `frontend/`. Nothing here imports from
or modifies the hackathon pipeline.

## Read first

- [`PLAN.md`](./PLAN.md) — the master research plan (thesis statement,
  research questions, fusion-variant grid, timeline, success criteria).

## Folder map

| Path | Purpose | Contents |
|---|---|---|
| `PLAN.md` | Master plan | The thesis proposal |
| `experiments/` | Per-experiment results | Empty until experiments run |
| `notes/` | Working notes, decision log | Add `DECISIONS.md` here as choices are made |
| `references/` | Cited papers, summaries | PDFs not committed; summaries are |
| `datasets/` | Per-dataset download + index scripts | Not committed |
| `splits/` | Patient-aware splits as CSVs | Generated, not hand-edited |
| `models/` | Per-variant model files (V0–V11) | One module each |
| `configs/` | YAML configs per variant + seed | Generated by templates |

## What this folder is NOT

- **Not a continuation of the hackathon.** Hackathon checkpoints and
  CSVs are *not* reused. The hackathon split has 66% patient leakage;
  research starts with a clean lesion-aware split.
- **Not a deployment.** No FastAPI, no frontend, no ngrok. Deliverables
  are the model variants + the paper.
- **Not coupled to hackathon code.** If the hackathon backend changes,
  this folder is unaffected; if research changes, hackathon is unaffected.

## Workflow isolation guarantee

The hackathon backend at `backend/` is **frozen-in-time** for the demo
and must keep working unchanged. Research changes go only inside
`research/`. If a research utility ever needs hackathon code, it imports
from `backend/` — never the other way around.
